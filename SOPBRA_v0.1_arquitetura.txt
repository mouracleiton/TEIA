O QUE É

SOPBRA é um metodologia que codifica COMO a política brasileira funciona
na PRÁTICA -- não como deveria funcionar segundo a Constituição, mas
como realmente opera: quem tem poder, como fluxa o dinheiro, onde estão
os pontos de alavancca, e como navegar o sistema para produzir mudança.

Não é um partido. Não é uma ONG. Não é um manifesto.
É um SISTEMA OPERACIONAL -- um conjunto de módulos, APIs, processos
e ferramentas que qualquer ator (ativista, jornalista, parlamentar,
servidor, empreendedor) pode usar para operar dentro do sistema
político brasileiro com eficácia máxima.

ANALOGIA: se a política brasileira é um computador legado (lento,
capturado, cheio de bugs), SOPBRA é o manual do kernel + as APIs
+ os drivers que permitem você rodar seus programas nele.


================================================================================
ARQUITETURA DO SISTEMA
================================================================================

SOPBRA tem 7 camadas, inspiradas na arquitetura de sistemas operacionais:

------------------------------------------------------------------------
CAMADA 0 — KERNEL (as regras reais do jogo)
------------------------------------------------------------------------
A constituição informal do poder no Brasil. Como o sistema realmente
funciona por baixo da Constituição formal.

Módulos do kernel:
  K1 — REGRA DA PAUTA: quem controla o que vota, controla tudo
  K2 — REGRA DO ORÇAMENTO: quem controla a emenda, controla o voto
  K3 — REGRA DO JUROS: quem controla a dívida, controla o Tesouro
  K4 — REGRA DA BANCADA: o eixo é setorial, não ideológico
  K5 — REGRA DO PATRIMÔNIO: política é negócio familiar, não serviço público
  K6 — REGRA DO JUDICIÁRIO: o STF é o supra-poder quando acionado
  K7 — REGRA DA OPACIDADE: o que não é visto, não é cobrado

------------------------------------------------------------------------
CAMADA 1 — DRIVERS (interfaces com cada Poder)
------------------------------------------------------------------------
Como se conecta com cada instituição do Estado:

  D1 — DRIVER EXECUTIVO (Presidência, Ministérios, Casa Civil)
  D2 — DRIVER LEGISLATIVO (Câmara, Senado, comissões)
  D3 — DRIVER JUDICIÁRIO (STF, STJ, CNJ, Justiça Federal)
  D4 — DRIVER MINISTÉRIO PÚBLICO (PGR, MPF, MPE)
  D5 — DRIVER TRIBUNAL DE CONTAS (TCU, TCEs)
  D6 — DRIVER CGU (Controladoria-Geral da União)
  D7 — DRIVER BANCO CENTRAL (Copom, CMN, regulação)
  D8 — DRIVER TSE (justiça eleitoral)
  D9 — DRIVER ESTADOS (governos estaduais, assembleias)
  D10 — DRIVER MUNICÍPIOS (prefeituras, câmaras municipais)

------------------------------------------------------------------------
CAMADA 2 — APIs (como operar cada driver)
------------------------------------------------------------------------
Para cada driver, um conjunto de ENDPOINTS (ações possíveis):

  API EXECUTIVO:
    POST /portaria → cria norma sem Congresso
    POST /decreto → altera regulamentação
    POST /medida-provisoria → força pauta (art. 62 CF)
    GET /audiencia → solicita reunião com ministro
    POST /ppa → insere prioridade no Plano Plurianual

  API LEGISLATIVO:
    POST /pl → apresenta projeto de lei
    POST /emenda → propõe emenda a PL
    POST /requerimento-cpi → solicita CPI
    GET /pauta → verifica o que está em pauta
    POST /audiencia-publica → convoca autoridade
    GET /votacao-nominal → verifica como cada parlamentar votou

  API JUDICIÁRIO (STF):
    POST /adi → Ação Direta de Inconstitucionalidade
    POST /adpf → Arguição de Descumprimento de Preceito Fundamental
    POST /amicus-curiae → ingressa como amigo da corte
    POST /re → Recurso Extraordinário

  API MPF:
    POST /representacao → documenta ilícito
    POST /acao-civil-publica → coleta dano ao erário
    GET /parecer → solicita manifestação

  API TCU:
    POST /representacao → documenta irregularidade
    GET /auditoria → solicita fiscalização
    POST /acordao → provoca decisão

  API CGU:
    POST /denuncia → canal de denúncia
    GET /auditoria → solicita auditoria em município
    POST /lai → pedido de acesso à informação

  API BCB/CMN:
    POST /consulta-publica → sugere regulamentação
    GET /estatisticas → acessa dados (SGS, REB)
    POST /parecer-tecnico → submete estudo

  API TSE:
    GET /prestacao-contas → acessa doações de campanha
    GET /ficha-limpa → verifica elegibilidade
    POST /representacao → denuncia irregularidade eleitoral

  API MUNICÍPIOS:
    GET /portal-transparencia → acessa gastos locais
    POST /conselho → participa de conselho municipal
    GET /licitacao → monitora licitações
    POST /orcamento-participativo → sugere prioridade

------------------------------------------------------------------------
CAMADA 3 — PROCESSOS (workflows para tarefas comuns)
------------------------------------------------------------------------
Sequências de ações para objetivos específicos:

  P1 — APROVAR LEI SEM CONGRESSO
    Portaria → decreto → MP → regulamentação
    (Quando usar: reformas que dependem só de executivo)

  P2 — BLOQUEAR LEI RUIM
    ADI no STF → liminar →audiência pública → mobilização
    (Quando usar: quando Congresso aprova algo prejudicial)

  P3 — EXPOR CAPTURA POLÍTICA
    Mapeamento → dossiê → narrativa pública → CPI → MPF
    (Quando usar: quando há evidência de captura documentada)

  P4 — INSTALAR CPI
    Coletar assinaturas → requerimento → votação → instalar
    (Quando usar: quando há esquema que precisa ser investigado)

  P5 — DIGITALIZAR SERVIÇO PÚBLICO
    Lei existe → CNJ/órgão regulador → petição → implementação
    (Quando usar: quando a lei está aprovada mas não implementada)

  P6 — REDUZIR SPREAD BANCÁRIO
    Dossiê BCB → consulta pública CMN → resolução → Open Finance
    (Quando usar: quando precisa reduzir custo de capital sem Congresso)

  P7 — EXIGIR TRANSPARÊNCIA DE EMENDAS
    LAI → Portal da Transparência → dashboard IA → STF ADPF 854
    (Quando usar: quando emendas são opacas)

  P8 — CRIAR COALIZÃO DE GANHADORES
    Mapear aliados → fórum → associação → co-financiamento
    (Quando usar: quando precisa contrabalançar lobby setorial)

  P9 — NEUTRALIZAR VETO PLAYER
    Mapear vulnerabilidades → escolher estratégia → executar
    (Quando usar: quando um ator bloqueia reforma)

  P10 — PRODUZIR CONTEÚDO DE PRESSÃO
    Dado verificado → roteiro → TikTok/YouTube/podcast → amplificação
    (Quando usar: quando precisa criar pressão pública)

------------------------------------------------------------------------
CAMADA 6 — SEGURANÇA (proteção contra captura)
------------------------------------------------------------------------
Como proteger o operador e o sistema:

  SEC-1 — ANONIMATO DE FONTE
    Toda denúncia baseada em fonte pública verificável
    NUNCA usar dados sigilosos ou vazados

  SEC-2 — PRESUNÇÃO DE INOCÊNCIA
    Correlação temporal ≠ causalidade
    Sempre citar: "documentado, não condenado"

  SEC-3 — PROTEÇÃO JURÍDICA
    Todo documento protocolável revisado por advogado OAB
    Amparo legal citado em cada afirmação

  SEC-4 — INTEGRIDADE DE DADOS
    Hash MD5 + SHA-256 em cada documento
    Versionamento Git com histórico imutável

  SEC-5 — PROTEÇÃO CONTRA RETALIAÇÃO
    Documentos públicos baseados em fontes públicas
    NUNCA incluir informação pessoal não pública

================================================================================
OS 7 PRINCÍPIOS DO KERNEL POLÍTICO BRASILEIRO
================================================================================

Estes são os axiomas -- as regras imutáveis que governam o sistema.

K1 — REGRA DA PAUTA
  Quem controla O QUE VOTA e QUANDO tem mais poder que quem vota.
  O veto mais poderoso não é o NÃO. É o silêncio (não pautar).
  IMPLICAÇÃO: para aprovar algo, não precisa convencer 257.
  Precisa convencer 1 (quem controla a pauta) ou criar custo de não pautar.

K2 — REGRA DO ORÇAMENTO
  Emenda parlamentar = moeda política. Quem tem emenda, tem voto.
  Reeleição de prefeito em cidade-cativo: 93-98%.
  IMPLICAÇÃO: para quebrar o sistema de captura, precisa quebrar o
  vínculo emenda-voto. Via: identificação nominal (ADPF 854) +
  Cartão de Emendas no TSE.

K3 — REGRA DOS JUROS
  A dívida pós-fixada cria alinhamento entre BC e rentistas.
  Cada ponto de Selic transfere R$ X bi do Tesouro para detentores.
  IMPLICAÇÃO: a prefixação (já em 47%) é o instrumento de libertação.
  Não precisa reduzir Selic — precisa reduzir SENSIBILIDADE à Selic.

K4 — REGRA DA BANCADA SETORIAL
  O eixo relevante não é esquerda/direita. É SETORIAL.
  PL e PSOL estão igualmente na bancada do funcionalismo.
  IMPLICAÇÃO: não tente convencer por ideologia. Tente por INTERESSE
  MATERIAL alinhado (quem ganha com a reforma vira aliado).

K5 — REGRA DO PATRIMÔNIO FAMILIAR
  Política no Brasil é negócio familiar, não serviço público.
  Famílias controlam estados por gerações (Sarney/MA, Lira/AL,
  Calheiros/AL, Braga/Aziz/AM, Ciro/Eliane/PI).
  IMPLICAÇÃO: para quebrar oligarquia, precisa de financiamento
  público de campanha + Cartão de Emendas que exponha o fluxo.

K6 — REGRA DO JUDICIÁRIO
  O STF é o supremo poder quando acionado corretamente.
  Pode derrubar leis (ADI), determinar transparência (ADPF),
  bloquear atos do executivo e legislativo.
  IMPLICAÇÃO: quando o Congresso se captura, o Judiciário é a
  válvula de escape. Usar ADPF e amicus curiae estrategicamente.

K7 — REGRA DA OPACIDADE
  O que não é visto, não é cobrado. O sistema funciona porque
  ninguém vê: o fluxo de emendas, o custo da Selic, o superfaturamento,
  o conflito de interesse, a drenagem financeira inter-regional.
  IMPLICAÇÃO: a primeira batalha é sempre pela VISIBILIDADE.
  Nota Técnica de Impacto Fiscal, IQD, Dashboard, Cartão de Emendas —
  todos são instrumentos de tornar o invisível visível.

================================================================================
CICLO DE OPERAÇÃO SOPBRA
================================================================================

Toda operação no sistema político brasileiro segue um ciclo de 5 fases:

FASE A — RECON (observação)
  Mapear o ambiente: quem tem poder, qual o status quo, onde estão
  os pontos de alavancca. Usar skills: mapeamento-veto-players,
  diagnostico-territorial.

FASE B — DIAGNÓSTICO (análise)
  Entender o problema: causa raiz, atores envolvidos, recursos em jogo.
  Usar skill: pipeline-dialetico, fact-check-economico.

FASE C — ESTRATÉGIA (planejamento)
  Escolher a rota de menor resistência: via executivo, legislativo,
  judiciário ou misto. Calcular probabilidade. Usar skill:
  simulacao-monte-carlo.

FASE D — EXECUÇÃO (ação)
  Acionar a API correspondente. Produzir o documento. Protocolar.
  Publicar. Pressionar. Usar skills: redacao-juridica-advocacy,
  producao-conteudo-advocacy.

FASE E — MONITORAMENTO (feedback)
  Acompanhar resultado. Medir impacto. Ajustar estratégia.
  Usar skill: sistema-protocolo para versionar mudanças.


================================================================================
EXEMPLO DE OPERAÇÃO COMPLETA
================================================================================

OBJETIVO: Reduzir spread bancário de 28-30 para 22-24 p.p.

FASE A — RECON:
  Mapear: spread de 28-30 p.p. (BCB SGS 20783)
  Decomposição: 84% custo, 16% margem (BCB REB 2023)
  Veto players: Hugo Motta (bloqueou tributação IFs), Febraban
  Aliados: Nubank, XP, Sicredi, CNI, Fiesp

FASE B — DIAGNÓSTICO:
  Causa raiz: spread é estrutural (Basileia + inadimplência + tributos)
  Não é só oligopólio. Competição reduz margem mas não custo.
  4 frentes necessárias: tributação, garantias, competição, dados

FASE C — ESTRATÉGIA:
  Via legislativo:bloqueada (Motta controle pauta, P=88% de veto)
  Via executivo: FACTÍVEL (resolução CMN, portaria BCB)
  Via judiciário: não aplicável
  Melhor rota: BCB/CMN para Frentes B (garantias) e D (dados) +
    executivo para Frente C (Open Finance)

FASE D — EXECUÇÃO:
  1. Dossiê técnico para BCB mostrando redução de inadimplência via
     registro digital de garantias (Frente B)
  2. Consulta pública CMN sobre Open Finance ampliado (Frente C+D)
  3. Via Haddad: portaria expandindo dados alternativos de crédito
  4. Conteúdo público: TikTok "O imposto privado de 30%"
  5. Coalizão: Fórum de Competição Financeira (Nubank+XP+Sicredi)

FASE E — MONITORAMENTO:
  Dashboard: spread médio mensal (BCB SGS 20783)
  Meta: 28-30 → 26-28 em 24 meses → 22-24 em 60 meses
  Alerta: se spread sobe ao invés de descer, estratégia falhou

O QUE O SOPBRA NÃO É
================================================================================

NÃO é um partido político. Não apoia candidato.
NÃO é ilegal. Todas as ações são via canais institucionais legais.
NÃO é hacker de sistema eleitoral. Não manipula votos.
NÃO é operação de inteligência. Usa apenas fontes públicas.
NÃO é violence. Não usa força física ou ameaça.
NÃO é paternalista. Não diz ao usuário o que pensar.

SOPBRA é um METODOLOGIA de navegação institucional. Como um GPS:
mostra o caminho, calcula a rota, alerta sobre obstáculos.
Mas não dirige o carro. O usuário decide o destino.
