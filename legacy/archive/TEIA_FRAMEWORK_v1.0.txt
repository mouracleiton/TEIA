O QUE É ESTE FRAMEWORK

O TEIA Framework é um sistema integrado que permite a qualquer ator
(analista, ativista, jornalista, servidor público, parlamentar, cidadão)
DIAGNOSTICAR como o poder funciona no Brasil, QUANTIFICAR quem lucra,
MAPEAR quem bloqueia mudança, SIMULAR estratégias de intervenção,
EXECUTAR ações concretas, e MEDIR resultados.

Não é teoria. É ENGENHARIA POLÍTICA aplicada.

Cada módulo foi testado em pipeline dialético, verificado contra fontes
primárias, e possui documentos de ação prontos para execução.

================================================================================
NÚCLEO — SOPBRA (SISTEMA OPERACIONAL POLÍTICO BRASILEIRO)
================================================================================

O SOPBRA é o kernel. Codifica COMO a política brasileira funciona
na prática — as regras reais do jogo, não as constitucionais.

------------------------------------------------------------------------
OS 9 AXIOMAS DO KERNEL
------------------------------------------------------------------------

K1 — REGRA DA PAUTA
  Quem controla O QUE VOTA e QUANDO tem mais poder que quem vota.
  O veto mais poderoso é o SILÊNCIO (não pautar).
  OPERAÇÃO: para mover a pauta, crie custo de não pautar.

K2 — REGRA DO ORÇAMENTO
  Emenda parlamentar = moeda política. Quem tem emenda, tem voto.
  OPERAÇÃO: quebre o vínculo emenda-voto via identificação nominal.

K3 — REGRA DOS JUROS
  A dívida pós-fixada alinha BC e rentistas.
  OPERAÇÃO: prefixar (47% já) reduz sensibilidade, não Selic.

K4 — REGRA DA BANCADA SETORIAL
  O eixo relevante é SETORIAL, não ideológico.
  OPERAÇÃO: identifique ganho material. Divida a bancada.

K5 — REGRA DO PATRIMÔNIO FAMILIAR
  Política é negócio familiar (Sarney, Lira, Calheiros, Braga).
  OPERAÇÃO: exponha via TSE + MPF + conteúdo.

K6 — REGRA DO JUDICIÁRIO
  O STF é supremo quando PROVOCADO corretamente.
  OPERAÇÃO: ADPF + amicus curiae estratégicos.

K7 — REGRA DA OPACIDADE
  O que não é visto não é cobrado.
  OPERAÇÃO: primeira batalha é SEMPRE pela VISIBILIDADE.

K8 — REGRA DA PACIÊNCIA ESTRATÉGICA
  Reforma leva 5-15 anos. O sistema exaure reformadores.
  OPERAÇÃO: ser SISTEMA, não pessoa. Versionar. Sobreviver.

K9 — REGRA DA JANELA
  Reformas só passam em janelas (pós-crise, pós-eleição, OCDE).
  OPERAÇÃO: fora de janela, acumule. Dentro de janela, execute.

2.2 PIPELINE DIALÉTICO (método de validação)
------------------------------------------------------------------------

Toda análise TEIA passa por 4 estágios antes de ser publicada:

  TESE (documento inicial)
    ↓
  FACT-CHECK (cada número verificado contra fonte primária)
    [FC OK] / [FC AJUSTAR] / [FC ERRO]
    ↓
  REFUTAÇÃO (ataque hostil por economista crítico)
    Identifica: aritmética falaciosa, inversão causal,
    ausência de teoria de mudança, limitações constitucionais
    ↓
  CONTRA-REFUTAÇÃO (defesa honesta)
    Para cada ataque: VENCE / PERDE / EMPATE
    Reformula o que caiu, mantém o que sobreviveu
    ↓
  SÍNTESE (documento final versionado)

REGRA: nenhum número entra sem fonte. Nenhuma proposta sem ter
sobrevivido a refutação.

CAMADA 3 — MAPEAMENTO
================================================================================

Como mapear atores, fluxos e sistemas de captura.

------------------------------------------------------------------------
3.1 MAPEAMENTO DE VETO PLAYERS (Framework Tsebelis + scores)
------------------------------------------------------------------------

Para cada ator político, atribuir 8 scores (0-10):

  PRG — Pragmatismo (flexibilidade ideológica)
  VJU — Vulnerabilidade Jurídica (processos ativos)
  VEL — Vulnerabilidade Eleitoral
  VFI — Vulnerabilidade Financeira/Pessoal
  POD — Poder institucional atual
  IDE — Ideologia/Crença genuína
  MUD — Histórico de mudança de posição
  ALI — Aliados institucionais

TIPOLOGIA RESULTANTE:
  VULNERÁVEL (VJU/VFI alto): fácil de mover por pressão
  PRAGMÁTICO (PRG alto, IDE baixo): fácil de cooptar por interesse
  IDEOLÓGICO (IDE alto, VJU baixo): só se move por argumento
  IRREMOVÍVEL (IDE alto, tudo baixo): não cooptável nem neutralizável

------------------------------------------------------------------------
3.2 SISTEMAS DE CAPTURA (3 eixos transversais)
------------------------------------------------------------------------

SISTEMA 1 — CAPTURA FINANCEIRA
  Quem: bancos top-5, Febraban, Banco Master, asset managers
  Como: juros da dívida (R$ 950 bi), spread (28-30 p.p.)
  Veto central: Motta (Câmara) + Alcolumbre (Senado) + Renan (CAE)

SISTEMA 2 — CAPTURA ORÇAMENTÁRIA
  Quem: Centrão, oligarquias estaduais, construtoras regionais
  Como: emendas de liderança (R$ 1,3 bi/ano, 96% em 4 partidos)
  Veto central: Lira + Coronel + Valdemar + Ciro

SISTEMA 3 — CAPTURA DA COMPLEXIDADE
  Quem: cartórios (325+ parlamentares), contadores, Sistema S
  Como: Custo Brasil (R$ 1,7 tri), conformidade tributária (1.501h/ano)
  Veto central: Zé Neto + André Figueiredo + L.C. Motta

------------------------------------------------------------------------
3.3 FLUXO DE DINHEIRO TERRITORIAL (Bomba de Drenagem)
------------------------------------------------------------------------

  N/NE paga impostos → governo federal
  ↓
  FPE/FPM/emendas → prefeituras (R$ 624 bi/ano)
  ↓
  Empresas (frequentemente do Sudeste) → bancos (Sudeste)
  ↓
  Crédito concentrado no Sudeste (53,6% PJ)
  ↓
  Remessa de lucros → Sudeste enriquece
  ↓
  N/NE permanece pobre (ciclo)

Métricas:
  Transferências: R$ 595,7 bi/ano
  Emendas: R$ 28,8 bi/ano (Gini territorial: 0,72-0,78)
  Drenagem privada: não mensurada (lacuna do BCB)

CAMADA 4 — SIMULAÇÃO
================================================================================

Como quantificar probabilidade de sucesso e escolher estratégia.

------------------------------------------------------------------------
4.1 MONTE CARLO (1.260.000 cálculos validados)
------------------------------------------------------------------------

Para cada veto player × estratégia, calcular:

  P_cooptação = f(PRG, MUD, IDE, ALI) + ajuste_estratégia
  P_neutralização = f(VJU, VEL, VFI, POD, ALI, IDE) + ajuste_estratégia
  P_combinada = 1 - (1-P_coopt)(1-P_neut)

9 ESTRATÉGIAS MODELÁVEIS:
  S1: CPI Master (P_média = 70,7%)
  S2: Narrativa pública (P = 63,7%)
  S3: Aliança Judiciário (P = 61,0%)
  S4: Coalizão ganhadores (P = 55,7%)
  S5: Pressão OCDE (P = 57,3%)
  S6: CPI + STF combinado (P = 75,0%)
  S7: Via Planalto/Lula (P = 87,1%) ← MAIS EFICAZ
  S8: Compensação setorial (P = 77,2%)
  S9: Via CNJ/Judiciário (P = 73,7%)

------------------------------------------------------------------------
4.2 DECISION TREE (rota de menor resistência)
------------------------------------------------------------------------

  Precisa de Congresso?
  ├─ NÃO (70% dos casos) → portaria / resolução / Judiciário
  └─ SIM (30%) → quem controla a comissão?
      ├─ ALIADO → tramitação normal
      ├─ NEUTRO → lobby técnico + coalizão
      └─ VETO PLAYER → Estado 6 (bloqueio)
          └─ Bypass: Via Planalto (P=87%) ou Judiciário (P=61%)

------------------------------------------------------------------------
4.3 THREAT MODEL (8 contramedidas)
------------------------------------------------------------------------

  Captura regulatória → OCDE + consulta pública
  Judicialização → amicus curiae + base legal
  Bloqueio de pauta → Via Planalto + narrativa
  Manobra orçamentária → STF (orçamento constitucional)
  Desinformação → fact-check imediato
  Cooptação reversa → código de conduta (não aceitar cargo)
  Intimidação jurídica → fonte pública = exercício regular
  Exaustão → sistema versionado sobrevive a pessoas

------------------------------------------------------------------------
4.4 COALITION BUILDER (5 camadas)
------------------------------------------------------------------------

  CAMADA 1 — ESTRUTURAIS (sempre a favor):
    Econômica: Tesouro, fintechs, cooperativas, CNI, MPF, CGU
    Ambiental: IBAMA servidores, ONGs, MapBiomas
    Tecnológica: desenvolvedores, comunidade open-source

  CAMADA 3 — NEUTROS MOBILIZÁVEIS:

OS 7 MECANISMOS DE ACCOUNTABILITY
================================================================================

Como fazer quem manuseia dinheiro público se comportar bem.

A equação: CUSTO do desvio deve ser > BENEFÍCIO do desvio.

HOJE: custo ≈ 0. Razão custo/benefício ≈ infinita a favor do desvio.

OS 7 MECANISMOS:

  M1 — TRANSPARÊNCIA EM TEMPO REAL
    IA detectando superfaturamento enquanto acontece
    Custo: R$ 1,5-2 mi

  M2 — CONSEQUÊNCIA RÁPIDA
    Lei dos 90 Dias: bens bloqueados em 90 dias após investigação
    Custo: R$ 0 (reorganização CNJ)

  M3 — RECOMPENSA POSITIVA
    Bônus de eficiência: municípios que melhoram indicadores
    ganham 5-10% adicional no FPM (fundo separado, métricas robustas)
    Custo: R$ 0 (realocação)

  M4 — RESPONSABILIZAÇÃO PESSOAL
    Gestor responde com bens próprios por obra não entregue
    Custo: legislativo

  M5 — CONTROLE SOCIAL
    App "Minha Cidade, Meu Dinheiro" com notificação ao cidadão
    Custo: R$ 2-3 mi

  M6 — REPUTAÇÃO POLÍTICA
    Cartão de Emendas no TSE: histórico de cada parlamentar
    Custo: R$ 0 (resolução TSE)

  M7 — PRESSÃO INTERNACIONAL
    OCDE como alavanca para boas práticas
    Custo: R$ 0,3-0,5 mi

AS 3 AÇÕES PRIORITÁRIAS (R$ 4-5 mi total):
  1. App "Minha Cidade" (R$ 2-3 mi)

================================================================================
CICLO DE OPERAÇÃO (5 FASES)
================================================================================

Todo uso do framework segue este ciclo:

FASE A — RECON
  "Em qual estado o sistema está?"
  → Rodar detector de estado (5 min)
  → Identificar janela de oportunidade
  → Skill: observação de fontes primárias

FASE B — DIAGNÓSTICO
  "Qual o problema e quem está envolvido?"
  → Identificar dimensão (econômica, institucional, etc)
  → Aplicar pipeline dialético
  → Skill: fact-check + pipeline-dialetico

FASE C — ESTRATÉGIA
  "Qual a rota de menor resistência?"
  → Mapear veto players e seus scores
  → Rodar Monte Carlo (probabilidade)
  → Escolher driver e endpoint
  → Skill: mapeamento-veto-players + simulacao-monte-carlo

FASE D — EXECUÇÃO
  "Qual documento/ação executar?"
  → Selecionar PKG apropriado ou produzir novo
  → Protocolar/publicar/mobilizar
  → Skill: redacao-juridica + producao-conteudo

FASE E — MONITORAMENTO
  "Funcionou?"
  → Medir KPIs
  → Versionar resultado
  → Ajustar estratégia se necessário
  → Skill: sistema-protocolo

PRINCÍPIOS ORIENTADORES DO FRAMEWORK
================================================================================

1. O dinheiro está. Falta direcionar.
   O Brasil não é pobre. É mal gerido.

2. Crescimento é NECESSÁRIO mas INSUFICIENTE.
   Sem reforma institucional, crescimento é temporário.

3. A economia é o sobressolo. A árvore tem 6 dimensões.
   Comece pelo sobressolo (já está 90% coberto).

4. A corrupção é vulnerabilidade, não força.
   Atores corruptos são MAIS fáceis de mover.

5. 70% das ações não precisam de Congresso.
   Via portaria, resolução ou Judiciário.

6. A primeira batalha é sempre pela VISIBILIDADE.
   Sem transparência, não há pressão. Sem pressão, não há reforma.

7. Não existe sistema que torne todos bons.
   Existe sistema que torna o mau comportamento caro.

8. O sistema político captura com dinheiro.
   O TEIA liberta com propósito. Propósito se renova, dinheiro se gasta.

9. Reformas só passam em janelas.
   Fora de janela: acumule. Dentro: execute.

10. O framework é um SISTEMA, não uma pessoa.
    Se um operador sai, outro entra. Documentação versionada garante
    continuidade. Git é a garantia de sobrevivência.

COMO USAR O FRAMEWORK (3 CENÁRIOS)
================================================================================

CENÁRIO 1 — "Descobri um esquema de corrupção"
  1. RECON: estado do sistema (há crise? → Estado 2)
  2. DIAGNÓSTICO: fact-check todos os números (skill 2)
  3. ESTRATÉGIA: mapear veto players envolvidos (skill 3)
  4. EXECUÇÃO: produzir dossiê (skill 1) + representação MPF (skill 5)
     + TikTok (skill 6) + pressionar CPI
  5. MONITORAMENTO: acompanhar investigação, versionar

CENÁRIO 2 — "Quero propor uma reforma de política pública"
  1. RECON: há janela? (pós-crise, pós-eleição, OCDE?)
  2. DIAGNÓSTICO: pipeline dialético completo (tese → FC → refutação)
  3. ESTRATÉGIA: Monte Carlo para escolher via (Planalto? STF? Congresso?)
  4. EXECUÇÃO: dossiê técnico (skill 5) + amicus curiae (skill 5)
     + documentário (skill 6) + coalizão de ganhadores
  5. MONITORAMENTO: KPIs + ajuste

CENÁRIO 3 — "Sou cidadão e quero fiscalizar meu município"
  1. RECON: verificar Portal da Transparência do município
  2. DIAGNÓSTICO: quanto recebeu em emendas? Quem executou?
  3. ESTRATÉGIA: App "Minha Cidade" + denúncia CGU
  4. EXECUÇÃO: postar no TikTok + pressionar vereadores
  5. MONITORAMENTO: acompanhar obra/serviço


================================================================================
VERSIONAMENTO E INTEGRIDADE
================================================================================

Cada documento TEIA tem:
  - Protocolo: TEIA-AAAA-NNN
  - Versão: vX.Y
  - Hash MD5 + SHA-256
  - Categoria (01-12)
  - Status (INTERNO / PENDENTE / PUBLICADO)

Git workflow:
  - Commits com mensagem estruturada
  - Tags em marcos (v1.0, v2.0, etc)
  - Branch main é a produção
  - Nada é apagado. Histórico imutável.

GitHub: https://github.com/mouracleiton/TEIA
